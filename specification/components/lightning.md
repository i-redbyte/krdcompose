# Lightning

## Общее описание

`Lightning` — это переиспользуемый визуальный компонент для Jetpack Compose, который рисует
анимированные электрические эффекты поверх composable-контейнера. Компонент поддерживает:

- автоматическую генерацию случайных молний
- запуск молнии по нажатию
- непрерывный plasma-эффект по долгому нажатию
- свечение, вспышку, shock ring, anchor ring и afterimage-эффекты
- переключение тем во время работы
- настройку через отдельный экран

Компонент разделён на два уровня:

- **screens** — экраны с применением компонента
- **ui/components/lightning** — переиспользуемая логика отрисовки, модели, математика и
  вспомогательные функции

## Публичный API компонента

Главная точка входа в компонент:

```kotlin
@Composable
fun LightningBackground(
    modifier: Modifier = Modifier,
    config: LightningConfig = LightningConfig(),
    theme: LightningTheme = LightningThemes.BlueStorm,
    content: @Composable BoxScope.() -> Unit
)
```

### Параметры

#### `modifier`

Обычный Compose `Modifier` для размеров, отступов, фона и позиционирования.

#### `config`

Определяет, какие эффекты включены и как ведёт себя система.

#### `theme`

Определяет цвета фона, ядра молнии, свечения, plasma-центра и вспышки.

#### `content`

UI-контент, который размещается поверх фона с молниями.

## `LightningConfig`

Управляет поведением компонента во время работы.

Поля:

- `enabled` — глобально включает или выключает компонент
- `autoRandomEnabled` — включает случайные молнии по таймеру
- `autoMinDelayMs` — минимальная задержка между случайными молниями
- `autoMaxDelayMs` — максимальная задержка между случайными молниями
- `flashEnabled` — включает экранную вспышку
- `flashMaxAlpha` — максимальная прозрачность вспышки
- `touchLaunchEnabled` — разрешает запуск молнии по нажатию
- `longPressPlasmaEnabled` — включает plasma-режим по долгому нажатию
- `shockRingEnabled` — включает shock ring при активации plasma
- `plasmaAnchorRingEnabled` — включает кольцо якоря вокруг точки plasma
- `plasmaAfterimageEnabled` — включает afterimage для plasma-молний

Пример:

```kotlin
val config = LightningConfig(
    enabled = true,
    autoRandomEnabled = true,
    autoMinDelayMs = 600,
    autoMaxDelayMs = 1800,
    flashEnabled = true,
    touchLaunchEnabled = true,
    longPressPlasmaEnabled = true,
    shockRingEnabled = true,
    plasmaAnchorRingEnabled = true,
    plasmaAfterimageEnabled = true
)
```

## Как работает экран настроек

Экран настроек собирает `LightningDemoState` из выбранной темы и переключателей.

Обычный сценарий работы:

1. пользователь открывает настройки
2. выбирает тему
3. включает или выключает эффекты
4. нажимает кнопку показа демо
5. собранное состояние сохраняется
6. экран демо получает это состояние и отрисовывает результат

То есть экран демо почти не содержит логики настройки и просто показывает переданное состояние.

## Архитектура отрисовки

`LightningBackground` использует многоуровневую структуру:

1. слой фонового цвета
2. слой canvas-отрисовки молний
3. слой обработки жестов
4. слой UI-контента поверх всех эффектов

Это важно, потому что если повесить `pointerInput` на тот же контейнер, который содержит кнопки,
жесты могут конфликтовать с UI.

Правильная идея такая:

```kotlin
Box {
    Canvas(...)
    Box(modifier = Modifier.matchParentSize().pointerInput(...))
    Box { content() }
}
```

Так кнопки остаются нажимаемыми, а фон продолжает реагировать на касания.

## Поведение во время работы

Компонент одновременно ведёт несколько процессов.

### Цикл времени кадра

Через `withFrameNanos` обновляется `timeMs`.

Это время используется для:

- расчёта возраста молнии
- анимации shock ring
- расчёта пульса plasma
- плавного затухания

### Цикл случайных молний

Если `autoRandomEnabled = true`, то:

- выбирается случайная стартовая точка внутри canvas
- выбирается случайное направление
- луч доводится до границы экрана
- строится ломаная линия молнии
- при необходимости создаются ветки
- молния добавляется в список активных

### Цикл plasma-режима

Если пользователь делает long press и включён `longPressPlasmaEnabled`, то:

- создаётся центр plasma в точке касания
- вокруг центра постоянно испускаются короткие plasma-разряды
- при движении пальца центр перемещается
- при отпускании пальца plasma выключается

### Очистка состояний

На каждом кадре:

- удаляются молнии, у которых закончился lifetime
- удаляются shock ring, у которых закончился lifetime

Это нужно, чтобы списки не росли бесконечно.

---

## Модель жестов

Компонент использует `pointerInput` и `awaitEachGesture`.

### Короткое нажатие

Если `touchLaunchEnabled = true`, то при первом касании:

- из точки касания запускается молния в случайном направлении до границы экрана

### Долгое нажатие

Если `longPressPlasmaEnabled = true`, то:

- измеряется длительность удержания
- после превышения таймаута долгого нажатия:
    - включается plasma-режим
    - появляется shock ring
    - начинается генерация plasma-молний
- пока палец движется, центр plasma обновляется
- при отпускании plasma выключается

## Подробное описание визуальных эффектов

## 1. Основная молния

Молния представляется как ломаная линия:

```kotlin
List<Offset>
```

Эта линия строится между начальной и конечной точкой с шумом в промежуточных вершинах.

Отрисовка идёт в три слоя:

1. внешний glow
2. расширенное полупрозрачное ядро
3. тонкое яркое центральное ядро

Так получается выразительный эффект электрического разряда без шейдеров размытия.

## 2. Ветки

Ветки — это вторичные ломаные линии, которые стартуют из случайных точек основной молнии.

Они:

- короче основной молнии
- отклоняются от главного направления
- рисуются тоньше и слабее по альфе

## 3. Экранная вспышка

Каждая видимая молния добавляет вклад в `flashStrength`.

Итоговая прозрачность вспышки рассчитывается с ограничением сверху:

```kotlin
alpha = min(flashMaxAlpha, flashAccumulator * flashMaxAlpha)
```

После этого рисуется полупрозрачный прямоугольник на весь экран.

## 4. Plasma-корона

Plasma-центр состоит из нескольких слоёв:

- мягкое внешнее свечение
- яркое внутреннее ядро
- дополнительное anchor ring
- короткие вращающиеся электрические лучи

## 5. Shock ring

Когда plasma активируется, создаётся расширяющееся кольцо.

Во времени оно:

- увеличивает радиус
- уменьшает прозрачность
- уменьшает толщину линии

## 6. Afterimage

Afterimage работает как визуальный след от недавних plasma-разрядов.

Для этого те же траектории перерисовываются более широким и более прозрачным слоем. Это создаёт
ощущение энергетической инерции и плотности разрядов.

# Математика компонента

Ниже — отдельное подробное описание математической части.

## 1. Система координат

Компонент работает в системе координат Compose canvas:

- начало координат `(0, 0)` находится в левом верхнем углу
- ось `x` направлена вправо
- ось `y` направлена вниз

Точка задаётся как:

```kotlin
Offset(x, y)
```

Вектор также хранится как `Offset`.

---

## 2. Направляющий вектор по углу

Чтобы пустить молнию в случайную сторону, угол переводится в единичный вектор.

Формулы:

\[
x = \cos(\theta)
\]

\[
y = \sin(\theta)
\]

Где:

- `θ` — угол в радианах
- результат затем нормализуется

Идея в коде:

```kotlin
Offset(cos(rad), sin(rad)).normalized()
```

Это даёт вектор длины 1.

---

## 3. Нормализация вектора

Для вектора `v = (x, y)` длина вычисляется так:

```text
|v| = sqrt(x^2 + y^2)
```

Нормализованный вектор:

```text
v_normalized = (x / |v|, y / |v|)
```

Это используется для:

- задания направления
- построения ответвлений
- добавления продольного шума вдоль основной оси

---

## 4. Линейная интерполяция

Чтобы получить промежуточные точки между началом и концом молнии, используется линейная
интерполяция.

Для двух точек `A` и `B`:

```text
L(t) = A + (B - A) * t
```

Где `t` лежит в диапазоне от `0` до `1`.

Это создаёт опорную прямую, по которой затем строится ломаная траектория.

В коде концептуально:

```kotlin
lerp(start, end, t)
```

---

## 5. Перпендикулярный вектор

Чтобы сделать молнию ломаной, каждая промежуточная точка сдвигается в сторону, перпендикулярную
основному направлению.

Если основной вектор:

```text
d = (x, y)
```

то один из перпендикулярных векторов:

```text
n = (-y, x)
```

После нормализации этот вектор используется как направление бокового отклонения.

---

## 6. Огибающая функция для шума

Если все точки смещать одинаково сильно, молния часто выглядит слишком ломано у самого начала и у
самого конца.

Поэтому шум масштабируется функцией:

```text
envelope(t) = sin(pi * t)
```

Свойства этой функции:

- `envelope(0) = 0`
- `envelope(1) = 0`
- максимум находится примерно посередине

Тогда поперечное смещение промежуточной точки можно описать как:

```text
delta_side = normal * randomSigned * jitter * sin(pi * t)
```

Это означает:

- в начале молния остаётся прикреплённой к старту
- в середине становится наиболее хаотичной
- ближе к концу снова стабилизируется

Именно это делает траекторию визуально более естественной.

---

## 7. Продольный шум

Кроме бокового отклонения добавляется небольшой шум вдоль основной оси.

Если `dir` — нормализованный основной вектор, то:

```text
delta_forward = dir * randomSigned * jitter * 0.15
```

Это помогает убрать ощущение слишком равномерной сегментации.

---

## 8. Построение ответвлений

Ответвления создаются из случайной точки основной молнии.

Берётся локальное направление основной линии и поворачивается на угол `alpha`, например от `28` до
`84` градусов.

Формула поворота вектора `(x, y)`:

```text
x' = x * cos(alpha) - y * sin(alpha)
y' = x * sin(alpha) + y * cos(alpha)
```

Так получается новое направление ветки.

Длина ветки берётся как доля длины основной молнии.

---

## 9. Пересечение луча с границами canvas

Если молния начинается внутри экрана и должна уйти к краю, используется луч:

```text
p(t) = start + direction * t,   t >= 0
```

Для границ прямоугольника вычисляются кандидаты по времени пересечения:

```text
tx = (x_border - start.x) / dir.x
ty = (y_border - start.y) / dir.y
```

Дальше выбирается минимальное положительное значение. Это и есть ближайшая граница, в которую
упрётся луч.

Так молния всегда доходит до края canvas.

---

## 10. Затухание по времени

У каждой молнии есть:

- `bornAtMs`
- `lifeMs`

Нормализованный возраст:

```text
t = age / life
```

После ограничения в диапазон `[0, 1]` вычисляется fade:

```text
fade = 1 - t
```

Этот коэффициент используется для:

- прозрачности ядра
- прозрачности glow
- вклада во вспышку

Из-за этого молния плавно исчезает.

---

## 11. Анимация shock ring

Радиус shock ring растёт линейно со временем:

```text
radius(t) = 20 + 120 * t
```

Прозрачность падает так:

```text
alpha(t) = 1 - t
```

Толщина линии тоже уменьшается по мере затухания. В итоге кольцо сначала выглядит мощным, а потом
быстро рассеивается.

---

## 12. Пульсация plasma

Plasma-эффект использует периодическую функцию(синусоидальную пульсацию):

```text
pulse = 0.82 + 0.18 * sin(2 * pi * phase)
```

Где `phase` — фаза, вычисленная по времени жизни plasma.

Эта пульсация меняет:

- яркость внутреннего ядра
- интенсивность glow
- яркость anchor ring

Так как синус — гладкая периодическая функция, плазма выглядит как пульсирующая, а не как случайно
мигающая.

---

## 13. Радиальные градиенты

Plasma-корона использует радиальные градиенты с центром в точке касания.

По сути это приближение поля энергии:

- максимальная яркость в центре
- мягкое затухание к краю

Это не физическая модель плазмы, но визуально работает очень хорошо, потому что глаз воспринимает
сочетание:

- яркого центра
- мягкого ореола
- коротких электрических выбросов

как энергетический объект.

---

## 14. Накопление экранной вспышки

Каждая молния даёт вклад:

```text
flashContribution = flashStrength * fade
```

Общий вклад:

```text
flashAccumulator = sum(flashContribution_i)
```

А итоговая прозрачность:

```text
flashAlpha = min(flashMaxAlpha, flashAccumulator * flashMaxAlpha)
```
То есть если молний несколько одновременно, экран вспыхивает сильнее, но всё равно не выходит за
заданный предел.

---

## Производительность

Компонент выглядит насыщенно, но остаётся достаточно лёгким, потому что не использует тяжёлую
физическую симуляцию.

Он опирается на:

- короткоживущие списки точек
- простую векторную арифметику
- стандартную Canvas-отрисовку
- регулярную очистку состояний
- отсутствие bitmap-постобработки
- отсутствие обязательных кастомных шейдеров

Тем не менее производительность зависит от:

- размера экрана
- количества одновременно живых молний
- частоты plasma-всплесков
- качества GPU устройства

### Если нужна оптимизация

Можно уменьшить:

- частоту случайных молний
- количество сегментов молнии
- количество веток
- ширину glow
- частоту plasma burst
- использование afterimage

```kotlin
    ) {

    }
}
```

## Важная деталь по обработке ввода

Правильная схема:

- canvas рисует эффекты
- отдельный полноэкранный gesture-layer ловит касания
- слой контента располагается поверх и остаётся интерактивным

Если `pointerInput` повесить на тот же контейнер, где живут кнопки, кнопки могут перестать корректно
нажиматься.
