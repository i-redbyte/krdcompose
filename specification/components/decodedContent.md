# DecodedContent

Composable-компонент для создания эффекта **декодирования данных**.

При раскрытии контента сначала отображается зашумлённый текст, который постепенно превращается в
реальный.

---

## ✨ Особенности

- Эффект `scramble -> reveal`
- Постепенное раскрытие символов
- Управляется состоянием `expanded`
- Настраиваемая скорость и количество шагов
- Поддержка кастомных палитр через `HackerListStyle`

## 🚀 Использование

```kotlin
DecodedContent(
    expanded = expanded,
    details = item.details,
    style = style
)
```

---

## 🧩 Параметры

| Параметр   | Тип               | Описание                              |
|------------|-------------------|---------------------------------------|
| `expanded` | `Boolean`         | Показывает или скрывает блок          |
| `details`  | `String`          | Исходный текст для декодирования      |
| `style`    | `HackerListStyle` | Палитра, размеры и параметры анимации |
| `modifier` | `Modifier`        | Дополнительный modifier               |

---

## ⚙️ Как это работает

1. При `expanded = true` блок становится видимым
2. Запускается генерация промежуточного scramble-текста
3. На каждом шаге увеличивается доля раскрытых символов
4. В конце показывается оригинальный текст

---

## 🔬 Визуальная схема

```text
███████  ->  H█L███  ->  HEL███  ->  HELLO
```

---

## 🎨 Настройка анимации

```kotlin
val style = HackerListStyles.MatrixGreen.copy(
    animations = HackerAnimationSpec(
        decodeSteps = 20,
        decodeStepDelayMillis = 40,
        decodedTextRevealDelayMillis = 900
    )
)
```

---

## 🎭 Что можно кастомизировать

- Набор шумовых символов
- Скорость раскрытия
- Количество шагов
- Цвет scramble и финального текста
- Фон панели и форму контейнера

---

## ⚠️ Важно

- Не нужно делать слишком много шагов, если элементов много
- Для эффекта лучше использовать `FontFamily.Monospace`
- Самый чистый результат получается на коротких и средних строках

---

## 💡 UX советы

Эффект особенно хорошо работает:

- при expand карточки
- при показе технических данных
- в логах, payload-блоках и diagnostic UI
